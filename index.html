Augmented Reality / Graphics in Javascript

A single repository for AR/Graphics/Mobile experiments using mainly Javascript technologies

A mixing of modified examples by other authors as well as some of my own


Some Examples:

  • AR: Cubesquitos - a demo for a simple interactive game demo (argonJS - user pose)
  • AR: Test for AR with natural (unsupervised) feature tracking (a-frame, jsfeat)
  • CV: Basic natural (unsupervised) feature tracking - 2d surface - (modifications of existing jsfeat example)
  • AR: Butteflies - Sarphati Series - (argonJS - user pose)
  • AR: Butteflies 2 - Sarphati Series - (threeJS, improved code)
  • AR: Simple Smoke Simulation - Sarphati Series - (threeJS)
  • AR: Butterflies in Smoke - Sarphati Series - (argonJS - user pose)
  • AR: Smoking Factory - Sarphati Series - (arJS with HIRO)
  • AR: Fireworks 2019 (arJS with custom marker)
  • GR: Extensible images over video (konvaJS)
  • GR: A simple interactive quote machine (p5JS)
  • CV: Test for unsupervised image feature recognition (trackingJS)

Currently Testing:

  • Location, Orientation and Motion
  • arJS - Sarphati

Code Evaluations:

  • chimney: exercise applying @rainner's code style combined with Stemkoski's approach to arJS [code]
  • dragablecubes: analysis of the interactive part of dragable cubes's argonJS sample [code]
  • aframe and 2dsurface_tracking: my modifications of @Eugene Zatepyakin's jsfeat orb feature dectector to fit an aframe example suggested by @gmarty [demo, code in index.html]
  • 2dsurface_tracking revised: an overhaul of @Eugene Zatepyakin's jsfeat orb feature dectector for better separation of concerns and for reusability following @rainner's code style [demo, code in index.html]
  • smoke: mixings of component (?) pattern by @Marco Biedermann for a smoke simulation [code]
  • aweJS example
  • butterflies: a revised threeJS script written by @soju22 (Kevin Levron) on codepen [code]
  • butterflies revised: an overhaul of @soju22 (Kevin Levron) codepen to resemble some aspects of @Marco Biedermann's code style and @rainner's code style as well as the @Eugene Zatepyakin approach to interfaces and utilities; goals were clearer separation of concerns and reusability [code]
